# PokeDesk

> 포켓몬 테마 집사형 알림 패널  
> 놀고 있는 안드로이드 폰을 책상 위 전용 상태 패널로 재활용하는 프로젝트

---

## 프로젝트 개요

PokeDesk는 실내 고정형 안드로이드 폰을 위한 캐릭터형 알림 패널이다.  
샤오미 14T를 책상 위에 세워두고, 포켓몬 캐릭터가 집사처럼 현재 상태를 안내한다.

이 프로젝트의 목적은 단순한 대시보드 구현이 아니다.  
시간, 디바이스 상태, 일정, 할 일, 외부 환경 데이터를 받아서  
포켓몬 캐릭터가 말풍선과 상태 변화로 반응하는 인터페이스를 만든다.

---

## 목표

- 놀고 있는 안드로이드 폰을 실내 전용 상태 패널로 재활용
- 일반 대시보드 대신 캐릭터형 UI 구성
- 일정, 할 일, 상태 이벤트를 짧고 직관적으로 전달
- 책상 위 세컨드 스크린 형태로 상시 실행
- 무료 API와 로컬 데이터만으로 구성

---

## 핵심 컨셉

PokeDesk는 숫자와 텍스트만 나열하지 않는다.  
포켓몬 캐릭터가 현재 상태를 해석해서 집사처럼 안내한다.

출력 예시:

- 일정 시간 입력 없음 → 수면 상태 전환
- 특정 시간 도달 → 오늘 할 일 확인 요청
- 충전 완료 → 회복 완료 메시지 출력
- 날씨/미세먼지 악화 → 경고 메시지 출력
- 장시간 미완료 할 일 존재 → 게으름 디버프 부여

## 주요 기능

### 1. 캐릭터형 상태 출력
포켓몬 캐릭터 상태로 알림을 표현한다.

- idle
- sleep
- focus
- warning
- healing
- event

### 2. 집사형 말풍선 알림
상황별 짧은 문장을 출력한다.

- "오늘 할 일을 아직 확인하지 않았어."
- "공기 상태가 좋지 않아. 경고 상태로 전환할게."
- "충전이 완료됐어. 회복이 끝났어."
- "오랫동안 입력이 없어. 잠든 상태로 전환할게."

### 3. 일정 / 할 일 브리핑
외부 일정과 할 일 데이터를 받아 요약해서 보여준다.

- 오늘 일정 개수
- 남은 할 일 개수
- 마감 임박 항목
- 장시간 미완료 항목

### 4. 디바이스 상태 반영
안드로이드 폰 상태를 캐릭터 상태에 반영한다.

- 배터리
- 충전 여부
- 화면 활성 상태
- 유휴 시간
- 네트워크 상태

### 5. 외부 환경 반영
외부 API 데이터를 받아 메시지와 상태에 반영한다.

- 날씨
- 미세먼지
- 시간대

## 왜 샤오미 14T인가

이 프로젝트에서 샤오미 14T는 개발 장비가 아니라 전용 실행 디바이스다.

역할:

- 책상 위 상시 실행 패널
- 캐릭터 UI 출력 단말
- 터치 입력 장치
- 충전 독 위 실내 고정 화면

PC에서 개발한 뒤 안드로이드 폰으로 배포하는 구조로 사용한다.

## 시스템 구조

```text
[외부 API / 로컬 데이터]
- 일정
- 할 일
- 날씨 / 미세먼지
- 디바이스 상태
- 포켓몬 데이터

        ↓

[FastAPI Backend]
- 상태 해석 엔진
- 시간대 판단
- 유휴 상태 판단
- 경고/회복 상태 계산
- 말풍선 메시지 선택
- 로컬 DB 저장

        ↓

[Frontend UI]
- 포켓몬 캐릭터 출력
- 말풍선 알림
- 상태 아이콘
- 오늘의 브리핑 패널
```

## PokeAPI 활용 방식

PokeDesk는 PokeAPI를 배틀용으로 사용하지 않는다.

캐릭터형 알림 패널에 필요한 정보만 사용한다.

사용 범위:

- 포켓몬 이름
- 타입
- 스프라이트 이미지
- 캐릭터 표현용 메타데이터

캐릭터 선택 규칙 예시:

- 경고 상태 → 독 / 고스트 / 악 타입 우선
- 회복 상태 → 풀 / 물 / 노말 타입 우선
- 집중 상태 → 에스퍼 / 전기 / 강철 타입 우선
- 수면 상태 → 노말 / 에스퍼 계열 우선

### 기본 상태

| 상태 | 의미 |
| --- | --- |
| idle | 기본 대기 상태 |

| 조건 | 상태 |
| --- | --- |
| 일정 시간 이상 입력 없음 | sleep |
| 배터리 낮음 | warning |
| 충전 중 | healing |
| 일정 또는 할 일 발생 | event |
| 사용자 활동 중 | idle / focus |

## 실행 방법

### [1단계] Windows에서 FastAPI 서버 실행 확인

#### 1. 가상환경 생성

```
python -m venv venv
```

#### 2. 가상환경 활성화

```
venv\Scripts\activate
```

#### 3. 패키지 설치

```
pip install -r requirements.txt
```

#### 4. 설치 확인

```
pip list
```

#### 5. 서버 실행

```
python -m app.main
```

#### 6. 접속 확인

```
http://127.0.0.1:8000
```

### [2단계] Termux 배포 및 실행

#### 1. Termux 기본 설정
```bash
pkg update && pkg upgrade -y
pkg install python git termux-api -y
pkg install termux-api
termux-setup-storage
```
#### 2. 프로젝트 복사
```bash
git clone https://github.com/seoheejung/pokedesk.git ~/pokedesk
cd ~/pokedesk

touch app/__init__.py
touch app/services/__init__.py
touch app/repositories/__init__.py
touch app/models/__init__.py
```

#### 3. 실행 권한 부여
```bash
chmod +x termux_setup.sh
chmod +x termux_start.sh
```

#### 4. 의존성 설치
```bash
./termux_setup.sh
```

#### 5. 서버 실행

```bash
./termux_start.sh
```
#### 6. 브라우저 접속

```bash
http://127.0.0.1:8000
```

#### 7. 상시 패널처럼 사용하는 방법

- 브라우저에서 PokeDesk 접속
- 전체화면 또는 홈 화면 추가 사용
- 충전기에 연결한 상태로 책상 위 고정
- 화면 꺼짐 시간은 Android 설정에서 별도 조정

#### 8. 종료 방법

Termux 실행 화면에서:

```bash
Ctrl+ C
```


#### 2. 브라우저에서 다시 접속

### ⚠️ 환경별 의존성 파일
| 파일                        | 용도               |
| ------------------------- | ---------------- |
| `requirements.txt`        | Windows / 로컬 개발용 |
| `requirements-termux.txt` | Termux 실행용       |


### ⚠️ 환경별 배터리 처리 방식
| 환경              | 방식                    |
| --------------- | --------------------- |
| Windows / Linux | psutil                |
| Termux          | termux-battery-status |
> psutil은 Termux에서 사용 불가

### ⚠️ 주요 주의사항
| 항목            | 내용                           |
| ------------- | ---------------------------- |
| 실행 위치         | 반드시 프로젝트 루트에서 실행             |
| 포트            | 기본 8000 사용                   |
| Termux 배터리 조회 | `termux-api` 필요              |
| 로컬 개발         | `requirements.txt` 사용        |
| Termux 실행     | `requirements-termux.txt` 사용 |
| 네트워크 접속       | PC에서 폰에 접근할 경우 동일 Wi-Fi 필요   |

## 개발 원칙

- 폰은 전용 실행 단말로 본다
- UI는 항상 켜둔 상태를 전제로 단순하게 유지한다

- 무료 API만으로 동작 가능해야 한다
